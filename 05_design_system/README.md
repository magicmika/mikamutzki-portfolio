# 05_design_system

Durable, Git-versioned memory for the development of Foundation V2.

## Source roles

- Documents explain why decisions were made.
- Figma provides visual evidence and implementation artifacts.
- zeroheight is a temporary, curated publication prototype, not a source of truth.

## Status vocabulary

- `RESEARCH` — sourced input, not a project decision.
- `EXPLORATION` — intentionally open investigation.
- `CANDIDATE` — promising, awaiting application tests.
- `APPROVED` — validated and confirmed by Mika.
- `REJECTED` — tested and deliberately not pursued.

## Contents

- `11_FOUNDATION_V2_DECISION_LOG.md` — chronological decisions and rationale.
- `12_FOUNDATION_V2_PRINCIPLES.md` — current principles and open candidates.
- `13_FOUNDATION_V2_TEST_LOG.md` — hypotheses, Figma tests and findings.
- `14_FOUNDATION_V2_CHANGELOG.md` — concise history of changes.
- `research/` — Deep Research brief and report.
- `exports/` — selected evidence for GitHub, case study and zeroheight.

## Working rule

No token, component or Foundation rule becomes `APPROVED` merely because it exists in Figma or zeroheight. Approval requires a documented decision and appropriate application tests.
